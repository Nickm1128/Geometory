# M1-P00 Requirements — Workflow, Baseline, And Tooling

- `M1-P00-R01` **Pillar 6 — Vertical-slice discipline:** repository state must support autonomous, resumable, dependency-ordered work with explicit evidence and blockers.
- `M1-P00-R02` **Pillar 5 — Deterministic core:** the pre-M1 prototype must be remotely recoverable and its known-green validation preserved before behavioral changes.
- `M1-P00-R03` **Pillar 4 — Mobile-first feel:** the physical Galaxy S24 is authoritative and a dedicated emulator must closely match its logical test profile.
- `M1-P00-R04` **Pillars 4 and 6:** deterministic direct-to-screen visual fixtures and a separate QA package must enable repeatable UI inspection without normal game navigation or production hooks.
- `M1-P00-R05` **Sustainable delivery:** Godot, Android SDK, CI, project skills, and documentation must agree on current supported versions and commands.

These requirements implement the approved M1 plan; they do not expand its deferred scope.
