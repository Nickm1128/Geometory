# M1-P06 Requirements — Certification And Closeout

- `M1-P06-R01` **All six pillars:** one integrated 10–20 minute Alpha Medium human-vs-bot loop must be stable, readable, strategically meaningful, deterministic, and polished on the target phone.
- `M1-P06-R02` **Lifecycle quality:** clean install, upgrade, cold launch, background/process resume, repeated matches, win/loss, and last-match replay must work without data loss or crash.
- `M1-P06-R03` **Android quality:** normal and QA packages must have inspected API/ABI/permission manifests, acceptable interaction performance, reproducible hashes, and retained certification evidence.
- `M1-P06-R04` **Delivery integrity:** every authority document, phase tracker, blocker, skill, artifact, Git ref, remote checkpoint, and final report must agree before M1 tags are published.
- `M1-P06-R05` **Human gate:** objective automation precedes one explicit user playtest for subjective fun and feel; no automated claim substitutes for that sign-off.
